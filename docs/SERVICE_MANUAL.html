1. 서비스 개요

Stock-AI는 다음 3개 앱으로 구성된 멀티서비스 플랫폼이다.

  1. Frontend: React 기반 사용자 웹 UI
  2. Backend: Spring Boot 기반 인증/도메인/API 서버
  3. AI Service: FastAPI 기반 분석/수치 계산/리포트 엔진

핵심 사용자 가치:

  1. 종목 분석 자동화 (기술적/기본적/뉴스)
  2. 전략 검증 (백테스트)
  3. 포트폴리오 관리 (보유, 실적 일정, 리스크 분석)
  4. 모의투자 및 AI 챗봇 보조

2. 사용자 기능 사용법

2.1 가입/로그인/인증

사용 경로:

  1. 회원가입: /register
  2. 로그인: /login
  3. 2FA 사용 시: 로그인 후 6자리 OTP 또는 백업 코드 입력

유의사항:

  1. 로그인 세션 만료/비활성 종료 시 로그인 화면에서 사유 메시지가 표시된다.
  2. Refresh Token 기반 재인증이 동작하며, 로그아웃 시 서버 토큰 폐기 후 로컬 토큰이 정리된다.

2.2 대시보드 (/dashboard)

주요 사용 흐름:

  1. 빠른 분석 위젯에서 티커 입력 후 분석 상세 페이지로 이동
  2. 관심종목/최근 분석/시장 개요 위젯으로 핵심 상태 파악
  3. 사용자화 버튼으로 위젯 표시/숨김 및 순서 변경

팁:

  1. 자주 보는 위젯만 노출하면 첫 화면 탐색 시간이 줄어든다.
  2. 대시보드 구성은 브라우저 로컬에 저장된다.

2.3 분석 (/analyze)

주요 사용 흐름:

  1. 티커/시장/리스크 프로필 선택
  2. 분석 실행 후 인사이트/추천/근거 확인
  3. 필요 시 업데이트로 최신 데이터 재조회
  4. 상세 보기에서 근거/차트/리포트 생성으로 확장

핵심 확인 포인트:

  1. 추천 점수와 confidence를 같이 본다.
  2. fundamentals와 news ratio를 함께 확인해 단일 지표 편향을 피한다.

2.4 백테스트 (/backtest)

주요 사용 흐름:

  1. 전략/자본/수수료 입력 후 실행
  2. 결과에서 총수익률, CAGR, Max Drawdown, Sharpe 확인
  3. 히스토리 목록에서 과거 실행 결과 재확인

해석 팁:

  1. 총수익률만 보지 말고 최대낙폭/샤프를 같이 본다.
  2. 동일 전략이라도 티커/기간에 따라 결과가 크게 달라질 수 있다.

2.5 관심종목 (/watchlist)

주요 사용 흐름:

  1. 종목 추가
  2. 태그 생성/할당으로 그룹 관리
  3. 메모를 기록해 투자 가설/리스크 포인트 축적
  4. 항목 단위 인사이트 업데이트

실무적 활용:

  1. 태그로 테마(예: 반도체, 배당, 경기민감) 분류
  2. 메모 필드에 매수/매도 기준을 남겨 회고 품질 개선

2.6 포트폴리오 (/portfolio)

주요 사용 흐름:

  1. 포트폴리오 생성
  2. 보유 포지션 추가(티커/시장/수량/매수가/매수일)
  3. 포트폴리오 상세에서 손익/비중/요약 확인
  4. 필요 시 리밸런싱 기능 활용

2.7 배당 캘린더 (/dividends)

주요 사용 흐름:

  1. 포트폴리오 선택
  2. 예정/최근 배당 이벤트 확인
  3. 지급 일정과 규모를 포트폴리오 현금흐름 관점에서 점검

2.8 실적 캘린더 (/earnings)

주요 사용 흐름:

  1. 포트폴리오 선택
  2. 조회 기간(30/60/90/180일) 선택
  3. 예정 실적 발표일, EPS 예상/실제 확인

활용 팁:

  1. 실적 전후로 변동성 확대 가능성이 높아 포지션 크기/손절 기준을 사전 점검한다.

2.9 리스크 대시보드 (/risk-dashboard)

주요 사용 흐름:

  1. 포트폴리오 선택
  2. 조회 기간(6개월/1년/15개월) 선택
  3. 핵심 리스크 지표 확인:
    • 연환산 변동성
    • 최대 낙폭
    • VaR 95%
    • Expected Shortfall 95%
    • Sharpe, Beta, 분산 점수, 집중도
  4. 시계열 차트(Drawdown/롤링 변동성)로 구간 리스크 확인

해석 팁:

  1. 집중도(최대 비중)가 높으면 단일 종목 이벤트 리스크가 커진다.
  2. VaR와 ES를 함께 봐서 꼬리 리스크를 확인한다.

2.10 스크리너 (/screener)

주요 사용 흐름:

  1. 시장/밸류에이션/수익성 조건 설정
  2. 결과 목록에서 지표 비교
  3. 후보 종목을 분석/관심종목으로 이동

2.11 상관관계 (/correlation)

주요 사용 흐름:

  1. 비교 티커 입력
  2. 상관계수 및 보조 지표 확인
  3. 포트폴리오 분산 효과가 있는 조합 탐색

2.12 모의투자 (/paper-trading)

주요 사용 흐름:

  1. 시장(US/KR) 선택
  2. 가상 주문(매수/매도, 수량) 실행
  3. 잔고/보유/주문 이력 확인
  4. 필요 시 계좌 초기화

주의사항:

  1. 모의투자는 학습용이며 실제 체결/슬리피지와 차이가 있다.

2.13 AI 챗봇 (/chatbot)

주요 사용 흐름:

  1. 질문 입력
  2. 대화별 문맥 유지
  3. 필요 시 새 대화 생성/기존 대화 삭제

권장 질문 방식:

  1. "어떤 지표를 왜 봐야 하는지"처럼 목적+조건을 함께 입력하면 응답 품질이 높다.

2.14 사용량 대시보드 (/usage)

주요 사용 흐름:

  1. 기간(7/30/90/180/365일) 선택
  2. 호출량/캐시적중률/오류 상위 항목 확인
  3. 데이터 내보내기/오래된 로그 정리 수행

2.15 설정 (/settings)

주요 사용 흐름:

  1. 2FA 활성화/비활성화
  2. 알림/요약 설정 변경
  3. 테마(다크/라이트), 언어(한국어/영어), 기본 시장 설정

3. 언어 모드 사용 가이드

지원 언어:

  1. 한국어(ko)
  2. 영어(en)

변경 방법:

  1. 설정 > 언어에서 변경
  2. 변경 즉시 UI 텍스트 반영

동작 원칙:

  1. 선택 언어 번역 키가 누락되면 한국어(ko)로 자동 폴백
  2. 한국어에도 없으면 키 문자열 표시(개발 검출용)

번역 품질 점검 체크리스트:

  1. 로그인/2FA/설정 알림 메시지 언어 일관성
  2. 대시보드 설정 패널 문자열 언어 일관성
  3. 실적 캘린더/리스크 대시보드 신규 화면 번역 여부
  4. 모의투자 주문 성공 메시지/주문일시 포맷 언어별 표시

4. 운영 기능 사용법

4.1 운영자가 주로 보는 엔드포인트

  1. Backend health: /actuator/health
  2. Backend metrics: /actuator/prometheus
  3. AI health: /health
  4. Nginx health: /nginx-health

4.2 실행/중지

프로덕션 유사 실행:

docker compose up -d --build
docker compose ps

중지:

docker compose down

4.3 로그 확인

전체 로그:

docker compose logs -f

서비스별 로그:

docker compose logs -f backend
docker compose logs -f ai
docker compose logs -f nginx

4.4 운영 관측

구성:

  1. Prometheus: 메트릭 수집
  2. Grafana: 시각화
  3. Sentry: 애플리케이션 에러 추적

핵심 지표:

  1. ai_client_requests_total
  2. ai_client_latency
  3. 인증 실패율/에러코드 추이
  4. 캐시 적중률 관련 지표

4.5 장애 대응 기본 절차

  1. X-Request-Id 확보
  2. 동일 requestId로 백엔드/AI 로그 추적
  3. AI 타임아웃/HTTP 오류 비중 확인
  4. 캐시/DB/외부 API 상태 점검
  5. 필요 시 설정값(timeout/retry) 보수 조정 후 재검증

관련 문서:

  1. docs/OPERATIONS_RUNBOOK.md
  2. docs/ERROR_CATALOG.md
  3. docs/DEPLOY_CHECKLIST.md

4.6 릴리스/배포 체크

배포 전:

  1. 필수 환경변수 설정 확인
  2. DB 마이그레이션(Flyway) 적용 가능 상태 확인
  3. CI 통과 여부 확인

배포 후:

  1. 헬스 체크
  2. 핵심 사용자 플로우 로그인/조회 점검
  3. 메트릭 이상 유무 점검

5. CI/E2E 운영 체크

CI 구성:

  1. Frontend 테스트/빌드
  2. Backend 빌드/테스트
  3. AI 문법검사/린트/테스트
  4. Docker Compose 기반 E2E 스모크 테스트

E2E 스모크가 확인하는 핵심 플로우:

  1. 회원가입/로그인/토큰갱신/로그아웃
  2. 보호 API 접근
  3. 포트폴리오 생성/조회
  4. 관심종목 add/list/remove
  5. 실적 캘린더/리스크 대시보드 호출

관련 파일:

  1. .github/workflows/ci.yml
  2. scripts/e2e_smoke.py

7. 자주 묻는 질문

Q1. 분석 결과가 비어 보입니다.
A1. 외부 데이터 소스 상태, 티커 유효성, 캐시 상태를 먼저 확인한다.

Q2. 리스크 대시보드가 빈 값으로 보입니다.
A2. 포지션/가격 데이터 부족 또는 조회 기간 데이터 부족일 수 있다. 보유 종목과 기간을 점검한다.

Q3. 실적 캘린더 이벤트가 없습니다.
A3. 선택 포트폴리오 종목의 향후 이벤트 데이터가 없는 경우 빈 결과가 정상이다.

Q4. 언어가 일부만 바뀝니다.
A4. 최신 빌드 반영 여부와 번역 키 누락을 확인한다. 본 코드베이스는 누락 시 한국어 폴백을 사용한다.
